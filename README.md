# Разработка программного комплекса для реализации интерактивных квестов на базе Telegram User-ботов и ИИ

Данный проект представляет собой программную платформу для создания иммерсивных текстовых симуляций в мессенджере Telegram. Основное отличие от классических решений — использование **Telegram Client API (MTProto)** вместо Bot API, что позволяет ИИ-агенту имитировать поведение реального пользователя.

## Технологический стек

*   **Backend:** Python 3.14, FastAPI, Telethon (MTProto библиотека).
*   **Orchestration:** n8n (Self-hosted).
*   **Database:** PostgreSQL (хранение состояний и истории).
*   **AI Models:** 
    *   **Gemini 3 Flash** (через OpenRouter) — логическое ядро.
    *   **OpenAI Whisper** — распознавание речи (STT).
    *   **ElevenLabs** — синтез эмоциональной речи (TTS).
*   **Infrastructure:** Docker, Docker Compose, Nginx.

## Ключевые особенности

1.  **Глубокая имитация человеческого поведения:**
    *   Динамическая задержка ответа, рассчитанная на основе длины текста.
    *   Визуальный статус «Печатает...» в процессе генерации.
    *   «Умное» прочтение сообщений: задержка появления синих галочек, если персонаж «был неактивен».
    *   Использование строчного регистра и намеренных опечаток в зависимости от уровня паники персонажа.

2.  **Динамическое управление состоянием (State Management):**
    *   Игровая логика не зашита в код, а управляется через JSON-объект в БД.
    *   ИИ-агент самостоятельно пересчитывает ресурсы (кислород, заряд батареи) и обновляет состояние мира.

3.  **Мультимодальность:**
    *   Персонаж принимает голосовые сообщения и отвечает на них эмоциональным голосом.
    *   **Акустическая постобработка:** в Python реализован каскад фильтров (Low Pass 1800Hz) и наложение звуковых слоев (песок, скрип дерева) для создания атмосферы «заточения».
    *   Генерация визуальных улик (фото), стилизованных под камеру старого мобильного телефона.

## Архитектура системы

Проект разделен на три независимых контура:
1.  **Transport Layer (Python):** API-шлюз, преобразующий HTTP-запросы от n8n в события MTProto и выполняющий DSP-обработку звука.
2.  **Logic Layer (n8n):** Визуально спроектированные сценарии, управляющие ИИ-агентами, парсингом данных и маршрутизацией.
3.  **Admin Layer (911 Terminal):** Управляющий бот для авторизации пользователей и мониторинга игровых сессий.

## Структура репозитория

*   `/telegram_api`: Исходный код Python-сервиса (FastAPI + Telethon).
*   `/n8n`: Экспортные файлы воркфлоу в формате `.json`.
*   `/system_promts`: Системные промты для AI-Agents.

## Модельный сценарий: «Погребенный заживо»

В качестве демонстрации возможностей платформы реализован сценарий, в котором пользователь выступает в роли диспетчера 911, пытающегося спасти контрактного водителя Пола Конроя, закопанного в гробу где-то в Ираке. 
*   **Условие поражения:** Критическая нехватка кислорода или разряд телефона.
*   **Условие победы:** Успешная верификация местоположения и ID сотрудника.
